express-app

alpha
templates@>=v1.2.3

Create a fully functional Express app using ESM modules.

caution

This template is in its alpha stage and will be evolving rapidly with new features and updates. Changes may include directory structures, Express app patterns, and more. We are also planning to support:

  • React
  • Modes (production, playground, etc.)
  • Databases (SQL, etc.)
  • And more...

We welcome your feedback! Feel free to share ideas for tools or frameworks you'd like to see supported, or any thoughts you have on the structure of the Express app and the libraries we use.

Submit feature request

Usageâ€‹

Usage
tps express-app <app-name>

# or in a directory

tps express-app path/to/dir/<app-name>
Creates
| - <app-name>/
    | node_modules/
        | - all libraries node modules...
    | - src
        | - config/
            | - constrants.js
        | - middlewares/
            | - error-handler.js
        | - routes
            | - index.js
        | app.js
    | - .env
    | - package.json
    | - package-lock.json
Example
  • default
  • long build path
tps express-app funny-gifs
tps express-app projects/funny-gifs

Check out our examples section to see detailed instructions on how to use this template.

This templates is a part of Templates library. If you've already installed Templates, you'll have instant access to this template, and you can disregard this command.

Copyâ€‹

If you like this template, but want to modify a few things use the copy command. It allows you to duplicate the template into your project and tailor it to your needs.

# if your not initialized run
tps init

# copy template
tps copy express-app

How to useâ€‹

This template generates a fully functional Express app. The name you provide to the CLI will be used for both the app's directory and the package.json name. To create the app in a specific directory, prepend the directory path to the name.

Example:

If you're building a new Express app for a restaurant called chickens-burgers, you would run the following command:

tps express-app chickens-burgers

# if you want this directory in say a directory called projects, then run the following command
tps express-app ./project/chickens-burgers

This will create a new directory called chickens-burgers, set up all the necessary files and directories for your Express app, and install the required packages using your preferred package manager.

|- funny-gifs/
    | node_modules/
        | - all libraries node modules...
    | - src
        | - config/
            | - constrants.js
        | - middlewares/
            | - error-handler.js
        | - routes
            | - index.js
        | app.js
    | - .env
    | - package.json
    | - package-lock.json

Once your template is created, you can cd into the directory and run the following command to start the app:

npm run start

# dev mode with nodemon
npm run dev

Typescriptâ€‹

templates@>=v1.2.5

Use the typescript option to create a express app in typescript. This option will change all js extensions to ts, add a tsconfig.json file, add some types to various areas of the codebase, and modify the package.json to support typescript.

Example:

If you're building a new Express app for a restaurant called chickens-burgers, you would run the following command:

tps express-app chickens-burgers --typescript

# if you want this directory in say a directory called projects, then run the following command
tps express-app ./project/chickens-burgers --typescript

This will create a new directory called chickens-burgers, set up all the necessary files and directories for your Express app, and install the required packages using your preferred package manager.

|- funny-gifs/
    | node_modules/
        | - all libraries node modules...
    | - src
        | - config/
            | - constrants.ts
        | - middlewares/
            | - error-handler.ts
        | - routes
            | - index.ts
        | app.ts
    | - .env
    | - package.json
    | - package-lock.json
    | - tsconfig.json

Once your template is created, you can cd into the directory and run the following command to start the app:

npm run build && npm run start

# Dev mode with nodemon + tsx
npm run dev
